# Cara Mia

Cara Mia is a shared art-exhibit website for long-distance collaboration. It has login/signup, draggable exhibit widgets, drawing canvases, word boxes, music search, and email-based sharing with viewer or collaborator permissions.

## Local Run

```bash
npm install
npm start
```

Open `http://localhost:3000`.

Local data is saved in `database/cara-mia.sqlite`. On Render, the same app uses the managed Postgres database from `DATABASE_URL`.

Optional Sticker search uses GIPHY when `GIPHY_API_KEY` is set. The single Sticker picker includes both still sticker-style assets and animated GIF-style assets; without that key, Cara Mia falls back to its built-in gothic recommendations.

## Render Deploy

This repo includes `render.yaml` for a Blueprint deploy:

- Web service name: `cara-mia`
- Expected Render URL: `https://cara-mia.onrender.com` if the service slug is available
- Database: managed Postgres named `cara-mia-db`
- Session secret: generated by Render
- Optional Sticker provider: add `GIPHY_API_KEY` in the Render service environment

Deployment needs the folder pushed to a GitHub, GitLab, or Bitbucket repository. Then open a Blueprint from that repo in the Render Dashboard and apply it.
